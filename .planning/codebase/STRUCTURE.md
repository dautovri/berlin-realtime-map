# Codebase Structure

**Analysis Date:** 2025-01-25

## Directory Layout

```
berlin-realtime-map/
├── BerlinTransportMap/          # Main application source code
├── BerlinTransportMap.xcodeproj/ # Xcode project files
├── BerlinTransportMap.xcworkspace/ # Xcode workspace
├── fastlane/                    # Build automation
├── scripts/                     # Build and utility scripts
├── docs/                        # Documentation
├── build/                       # Build artifacts
├── marketing/                   # Marketing assets
└── .planning/                   # GSD planning documents
```

## Directory Purposes

**BerlinTransportMap/:**
- Purpose: Contains all Swift source files for the iOS application
- Contains: .swift files, .entitlements, .xcprivacy, .xctestplan
- Key files: `BerlinTransportMapApp.swift`, `TransportMapView.swift`, `TransportService.swift`

**BerlinTransportMap.xcodeproj/:**
- Purpose: Xcode project configuration and build settings
- Contains: .pbxproj, .xcconfig files
- Generated: Partially generated by Xcode

**fastlane/:**
- Purpose: Automated build, test, and deployment pipelines
- Contains: Fastfile, configuration files
- Committed: Yes

## Key File Locations

**Entry Points:**
- `BerlinTransportMap/BerlinTransportMapApp.swift`: Main application entry point
- `BerlinTransportMap/ContentView.swift`: Root SwiftUI view

**Configuration:**
- `BerlinTransportMap/BerlinTransportMap.entitlements`: App entitlements and capabilities
- `BerlinTransportMap/PrivacyInfo.xcprivacy`: Privacy manifest

**Core Logic:**
- `BerlinTransportMap/TransportMapView.swift`: Main map view and state management
- `BerlinTransportMap/TransportService.swift`: Static transport data service
- `BerlinTransportMap/VehicleRadarService.swift`: Real-time vehicle data service
- `BerlinTransportMap/LocationManager.swift`: User location management

**Testing:**
- `BerlinTransportMap/BerlinTransportMap.xctestplan`: Test plan configuration

## Naming Conventions

**Files:**
- PascalCase for type names (TransportMapView.swift, TransportService.swift)
- camelCase for variables and functions

**Directories:**
- PascalCase for main app directory (BerlinTransportMap)
- lowercase for utility directories (scripts, docs)

## Where to Add New Code

**New Feature:**
- Primary code: `BerlinTransportMap/` directory alongside existing files
- Add new .swift files for views, services, or models

**New Component/Module:**
- Implementation: `BerlinTransportMap/` as flat structure
- Follow existing naming: [Feature]View.swift, [Feature]Service.swift

**Utilities:**
- Shared helpers: `BerlinTransportMap/` with descriptive names like ColorExtensions.swift

## Special Directories

**build/:**
- Purpose: Contains compiled binaries and intermediate build files
- Generated: Yes
- Committed: No (gitignored)

**.planning/:**
- Purpose: GSD planning and analysis documents
- Generated: By GSD commands
- Committed: Yes
